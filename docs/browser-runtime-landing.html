Route By Evidence

One URL. One best path.

Grasp is the route-aware AI Browser Runtime. It chooses the best web path before execution starts, then keeps work continuous across login, checkpoints, confirmed browser boundaries, human handoff, and resumed execution instead of collapsing into disposable tabs, brittle scripts, or headless one-site automation.

Proof of the runtime
entry(url, intent) inspect() request_handoff(...) mark_handoff_done() resume_after_handoff() continue()

Opening a page is not the proof. Working inside a confirmed visible browser context, surviving a human step, and continuing from evidence is the proof.

What the loop returns
  • entry(url, intent) can return the selected route, evidence, and fallback boundary
  • inspect() can surface handoff_required instead of pretending the task is done
  • resume_after_handoff() can return resumed_verified with continuation evidence
  • continue() can route the task forward without restarting the browser session
Where The Moat Compounds

The hard part is not opening pages.

Most browser automation breaks after login, after a checkpoint, or after a human needs to step in once. Grasp compounds around the parts that are hard to fake.

Continuity

Tasks survive login state, checkpoint pages, and context switching instead of restarting from scratch.

Verification

Actions are checked against actual page changes, not treated as success because a tool call returned.

Recovery

Humans can step in, agents can resume, and the same browser context keeps the task coherent.

What Changes

From automation wrapper to route-aware runtime.

Publicly, Grasp is a route-aware runtime for agents. If that runtime keeps compounding across real web work, it becomes the operating layer agents rely on.

Ordinary Browser Automation

Useful for getting one step done. Fragile when the task becomes real.

  • opens a page, then forgets the broader task
  • assumes actions worked because the call returned
  • breaks when login or handoff interrupts the flow
Grasp Runtime

Built for work that stays real after the first page load.

  • treats any real webpage as part of the same task model
  • keeps evidence, continuation, and recovery first-class
  • lets agents re-enter the same browser context instead of replaying
First Win

Three steps to feel the product.

The first win is not “it can open a page.” The first win is that the agent can enter a real page, choose a route, explain it, and stay inside the same runtime as the task gets harder.

1

Bootstrap

Run the local runtime and keep using the dedicated `chrome-grasp` profile.

npx grasp
2

Connect

Add Grasp to your AI client so the agent can use the same runtime surface.

claude mcp add grasp -- npx -y grasp
3

Run the loop

Ask the agent to check status, enter a real page with intent, then inspect, continue, and explain the route.

get_status
entry(url, intent)
inspect()
extract() or continue()
explain_route()
