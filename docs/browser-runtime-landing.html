The first win is not “it opened a page.” The first win is that the agent can choose a route, explain it, and stay inside the same runtime as the task gets real.

Why Grasp

The hard part is not opening a page.

Most browser automation falls apart after login, after a checkpoint, or after a human needs to step in once. Grasp compounds around the parts that are hard to fake.

Route

Choose the best path first. Public read, live session, form, workspace, or handoff are runtime decisions, not user memory tests.

Verification

Actions are judged by real page state change, not by the fact that a tool call returned without throwing.

Recovery

Human handoff is not a failure mode. It is part of the runtime and can resume from evidence instead of replaying everything.

Dedicated Browser

The agent works in one visible browser boundary instead of guessing across arbitrary windows and disposable sessions.

Proof

What proves this is a runtime.

A real task does not restart from scratch just because a human had to help once. The loop below is the smallest proof that Grasp is more than a page opener.

entry(url, intent) inspect() request_handoff(...) mark_handoff_done() resume_after_handoff() continue()
